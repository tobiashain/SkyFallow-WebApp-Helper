# Schedule & Timeline Web App

A browser-based tool for designing **NPC daily schedules** and **Dialogic timeline tables** for our Godot game.  
Exports JSON files that are converted into `.tres` resources – no JSON parsing at runtime.

## Pipeline

### Schedules

1. **Web App** → `npc_schedules.json`
2. **Godot Editor Script** `create_npc_schedules.gd` → `.tres` resources in `res://schedules/`
3. **Baking** `bake_npc_nav_data.gd` collects schedule resource paths from NPC nodes and bakes them into `npc_nav_baked.json`
4. **Runtime** `NPCNavLoader` loads the baked JSON and the referenced schedule resources

### Timeline Table

1. **Web App** → `npc_timeline_table.json`
2. **Godot Editor Script** `create_npc_timelines.gd` → `.tres` resources in `res://timelines/`
3. **Runtime** `DialogicManager` directly uses these timeline resources (no baking step)

## Editors

### NPC Schedules

Per-NPC list of **entries** evaluated in order (first matching entry wins). Each entry can have:

- A **condition** (day, weekday, flag, friendship, AND/OR/NOT).
- Multiple **time‑based points** with a scene, target node, and optional per‑point condition.

### Dialogic Timeline Table

Per-NPC list of **timeline entries** with a weight (random selection) plus conditions. Unmatched entries fall back to a randomly chosen **fallback list**. Conditions can include:

- `day_from` – earliest game day
- `hour_range` – in‑game hours [from, to]
- `seen_first` – timeline that must have been played first
- Arbitrary structured condition trees (same condition system as schedules)

## Input Files (drag & drop)

### `npc_nav_baked.json` (required)

Generated by `bake_npc_nav_data.gd`. Supplies:

- All NPC ids and their home scenes
- Valid target node paths per scene
- Scene names / ids

### `npc_schedules.json` or `npc_timeline_table.json` (optional)

Import a previously exported file to continue editing.

## Output Files

- `npc_schedules.json` – placed at `res://data/npc_schedules.json`
- `npc_timeline_table.json` – placed at `res://data/npc_timeline_table.json`

## Condition System (shared by both editors)

| Type                  | Fields                            | Description                           |
| --------------------- | --------------------------------- | ------------------------------------- |
| `DayCondition`        | `day` (int)                       | current day equals `day`              |
| `WeekdayCondition`    | `weekday` (int, 0=Sun…6=Sat)      | weekday matches                       |
| `FlagCondition`       | `flag_name` (string)              | game flag is set                      |
| `FriendshipCondition` | `npc_id`, `required_hearts` (int) | player has enough hearts with the NPC |
| `AndCondition`        | `sub_conditions`                  | all must be true                      |
| `OrCondition`         | `sub_conditions`                  | at least one must be true             |
| `NotCondition`        | `condition`                       | inverts a single condition            |

The timeline table editor also uses flattened special conditions (`DayFromCondition`, `HourRangeCondition`, `SeenFirstCondition`) for convenience; these are merged into the condition tree on export.

### Example schedule entry (JSON)

```json
{
  "id": "martha",
  "entries": [
    {
      "description": "Day 3 special",
      "entry_condition": {
        "type": "AndCondition",
        "sub_conditions": [
          { "type": "DayCondition", "day": 3 },
          { "type": "FlagCondition", "flag_name": "player_met_martha" }
        ]
      },
      "points": [{ "time": 800, "scene": 0, "target_node": "Farm/House/Bed" }]
    }
  ]
}
```

### Example timeline table entry (JSON)

```json
{
  "timeline_table": {
    "martha": [
      {
        "timeline": "martha_greet",
        "weight": 2,
        "condition": {
          "type": "AndCondition",
          "sub_conditions": [
            { "type": "DayFromCondition", "day_from": 5 },
            { "type": "HourRangeCondition", "hour_from": 8, "hour_to": 18 }
          ]
        }
      }
    ]
  },
  "fallback_table": {
    "martha": ["martha_idle"]
  }
}
```
